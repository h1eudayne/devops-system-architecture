# Hướng Dẫn Thực Hành: Assume Role Với AWS CLI

Tài liệu này hướng dẫn từng bước cách thực hiện **Assume Role** qua dòng lệnh (AWS CLI). Chúng ta sẽ gỡ bỏ toàn bộ quyền trực tiếp của một IAM User, sau đó cấu hình cho phép User đó giả lập (Assume) một Role có quyền lực cao hơn (`MyPowerRole`) và thực hiện thao tác tài nguyên thông qua thông tin xác thực tạm thời (temporary credentials).

---

## Các Bước Thực Hiện Chi Tiết

### Bước 1: Gỡ bỏ User khỏi Group để xóa bỏ quyền hạn cũ

Trước tiên, chúng ta cần gỡ bỏ tất cả các quyền trực tiếp của User để kiểm tra xem tính năng Assume Role hoạt động như thế nào khi bản thân User không có quyền hạn.

1. Đăng nhập vào **AWS Console** bằng tài khoản Admin.
2. Truy cập **IAM** > chọn **IAM users** > Chọn user `developer-01`.
3. Tại tab **Permissions**, tích chọn group `developer-group` và chọn **Remove**.
4. Xác nhận loại bỏ User ra khỏi nhóm.

![Gỡ bỏ User khỏi Group](../../../../images/aws/iam_assume_role_remove_group.png)

---

### Bước 2: Kiểm tra quyền hạn của User trên CLI (Mong muốn: Bị Từ Chối)

1. Mở terminal/dòng lệnh trên máy tính cá nhân của bạn (nơi đang được cấu hình bằng Access Key mặc định của `developer-01`).
2. Chạy lệnh kiểm tra danh sách S3 bucket:
   ```bash
   aws s3 ls
   ```
3. **Kết quả mong muốn**: Hệ thống sẽ trả về lỗi **AccessDenied** vì User `developer-01` hiện tại đã không còn quyền hạn nào sau khi bị gỡ khỏi Group ở Bước 1.
   ```text
   An error occurred (AccessDenied) when calling the ListBuckets operation: Access Denied.
   ```

---

### Bước 3: Tạo Role `MyPowerRole` và cấu hình Trust Policy

Chúng ta sẽ tạo một IAM Role mới có quyền hạn `PowerUserAccess` và chỉ định cấu hình Trust Policy chỉ cho phép duy nhất User `developer-01` có quyền giả lập (Assume) vai trò này.

1. Truy cập **IAM Console** > Chọn **Roles** ở menu bên trái > Chọn **Create role**.
2. Tại mục **Trusted entity type**, chọn **Custom trust policy**.
3. Điền cấu hình Trust Policy JSON dưới đây (thay thế `<Account_ID>` bằng Account ID AWS thực tế của bạn, ví dụ: `080641082881`):
   ```json
   {
       "Version": "2012-10-17",
       "Statement": [
           {
               "Effect": "Allow",
               "Principal": {
                   "AWS": [
                       "arn:aws:iam::<Account_ID>:user/developer-01"
                   ]
               },
               "Action": "sts:AssumeRole"
           }
       ]
   }
   ```

   ![Cấu hình Custom Trust Policy](../../../../images/aws/iam_assume_role_custom_trust.png)

4. Nhấn **Next**. Tại trang **Add permissions**, tìm kiếm và tích chọn chính sách quản lý: **PowerUserAccess** > Nhấn **Next**.

   ![Chọn PowerUserAccess](../../../../images/aws/iam_assume_role_add_permissions.png)

5. Tại trang **Name, review, and create**:
   - Đặt tên cho Role: `MyPowerRole`.
   - Kiểm tra lại cấu hình Trust Policy và Permissions > Chọn **Create role** ở góc dưới cùng bên phải.

   ![Đặt tên Role và Kiểm tra](../../../../images/aws/iam_assume_role_review_create.png)

6. Sau khi tạo thành công, giao diện quản lý Role sẽ hiển thị thông tin chi tiết cùng ARN của Role mới (ví dụ: `arn:aws:iam::080641082881:role/MyPowerRole`). Hãy sao chép ARN này để sử dụng cho bước tiếp theo.

   ![Tạo Role thành công](../../../../images/aws/iam_assume_role_created_dashboard.png)

---

### Bước 4: Thực hiện Assume Role qua AWS CLI

1. Chạy lệnh dưới đây từ terminal để thực hiện yêu cầu giả lập vai trò (thay thế `<Role_ARN>` bằng ARN của `MyPowerRole` vừa sao chép ở Bước 3):
   ```bash
   aws sts assume-role --role-arn "arn:aws:iam::<Account_ID>:role/MyPowerRole" --role-session-name cli-session
   ```
2. **Kết quả trả về**: AWS STS sẽ trả về một khối JSON chứa thông tin xác thực tạm thời:
   ```json
   {
       "Credentials": {
           "AccessKeyId": "ASIAXXXXXXXXXXXXXXXX",
           "SecretAccessKey": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
           "SessionToken": "FwoGZXIvYXdzEOb//////////...",
           "Expiration": "2026-06-04T15:15:00+00:00"
       },
       "AssumedRoleUser": {
           "AssumedRoleId": "AROAXxxxxxxxxxxxxxxx:cli-session",
           "Arn": "arn:aws:sts::<Account_ID>:assumed-role/MyPowerRole/cli-session"
       }
   }
   ```

---

### Bước 5: Cấu hình file credentials để sử dụng quyền tạm thời

1. Mở file credentials của AWS trên máy cá nhân:
   - Trên Windows: `C:\Users\<UserName>\.aws\credentials`
   - Trên Linux/macOS: `~/.aws/credentials`
2. Thêm cấu hình một profile mới tên là `[poweruser]` và điền các giá trị nhận được từ lệnh `assume-role` ở Bước 4 vào:
   ```ini
   [poweruser]
   aws_access_key_id = <AccessKeyId_ở_Bước_4>
   aws_secret_access_key = <SecretAccessKey_ở_Bước_4>
   aws_session_token = <SessionToken_ở_Bước_4>
   ```

   > [!WARNING]
   > Hãy chú ý điền đầy đủ cả 3 thông số. Khác với access key tĩnh, thông tin xác thực tạm thời từ dịch vụ STS **bắt buộc** phải có trường `aws_session_token` thì mới hoạt động được.

---

### Bước 6: Kiểm tra kết quả (KT)

1. Chạy lệnh liệt kê danh sách S3 bucket sử dụng profile `poweruser` vừa tạo:
   ```bash
   aws s3 ls --profile poweruser
   ```
2. **Kết quả**: Lần này bạn sẽ liệt kê thành công các S3 bucket hiện tại trong tài khoản mà không còn bị từ chối truy cập (Access Denied) nữa, vì profile `poweruser` đang mang quyền hạn của role `MyPowerRole` (có đính kèm chính sách `PowerUserAccess`).
