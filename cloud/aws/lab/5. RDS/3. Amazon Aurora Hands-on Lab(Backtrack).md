# Amazon Aurora Hands-on Lab (Aurora Cluster & Backtrack)

Bài thực hành này hướng dẫn bạn từng bước khởi tạo cụm cơ sở dữ liệu **Amazon Aurora (MySQL-Compatible)**, kích hoạt tính năng **Backtrack (Quay ngược thời gian)**, thực hiện các thao tác CRUD cơ bản, giả lập tình huống xóa nhầm dữ liệu và sử dụng Backtrack để khôi phục dữ liệu tức thì ngay trên Cluster đang chạy.

---

## Các bước thực hiện chi tiết

### Bước 1: Khởi tạo Aurora Cluster và kích hoạt Backtrack
1. Đăng nhập vào AWS Management Console, truy cập dịch vụ **RDS** -> chọn **Databases** -> nhấp **Create database**.
2. Thiết lập cấu hình khởi tạo cụm máy chủ:
   * **Choose a database creation method**: Chọn **Standard create**.
   * **Engine options**: Chọn **Aurora (MySQL Compatible)**.
   * **Templates**: Chọn **Dev/Test** (hoặc Production tùy nhu cầu thực tế).
   * **Settings**:
     * **DB cluster identifier**: Đặt tên cho cụm (ví dụ: `aurora-cluster-01`).
     * **Master username**: `admin`.
     * **Master password**: Thiết lập mật khẩu quản trị.
   * **Instance configuration**: Chọn dòng DB instance class phù hợp (ví dụ: `db.r5.large`).
   * **Availability & durability**: Chọn **Create an Aurora Replica/Reader node in a different AZ** (Để khởi tạo cụm gồm 2 instances: 1 Writer và 1 Reader).
   * **Backtrack (Cấu hình quay ngược thời gian)**:
     * Tích chọn **Enable Backtrack** (Kích hoạt tính năng Backtrack).
     * **Target backtrack window**: Thiết lập khoảng thời gian tối đa bạn muốn quay ngược dữ liệu (ví dụ: nhập `24` hours).
   * Click chọn **Create database** ở dưới cùng.

### Bước 2: Kiểm tra trạng thái Cluster và các DB Instances con
Sau khi quá trình khởi tạo hoàn tất và cụm chuyển sang trạng thái **Available**, bạn sẽ thấy cơ cấu các node như sau:

* **Cluster tổng (Regional cluster)**: `aurora-cluster-01` hiển thị trạng thái `Available` quản lý `2 instances`.
* **Writer Instance (Máy chủ Ghi)**: `aurora-cluster-01-instance-1` (Role: **Writer instance**, AZ: `us-east-1b`).
* **Reader Instance (Máy chủ Đọc)**: `aurora-cluster-01-instance-1-us-east` (Role: **Reader instance**, AZ: `us-east-1d`).

![Kiểm tra danh sách máy chủ trong Cluster](../../../../images/aws/rds_lab_aurora_step2_list.png)

### Bước 3: Thực thi CRUD và giả lập tình huống xóa dữ liệu
1. Copy địa chỉ **Writer Endpoint** của cụm `aurora-cluster-01`.
2. Mở DBeaver (hoặc Client bất kỳ) và thực hiện kết nối.
3. Tạo cơ sở dữ liệu `company`, tạo bảng `employees` và chèn dữ liệu mẫu (sử dụng script mẫu tại [employee_crud_slow_query.sql](../../templates/rds/employee_crud_slow_query.sql)):
   ```sql
   CREATE DATABASE IF NOT EXISTS company;
   USE company;
   CREATE TABLE employees ( ... );
   INSERT INTO employees ...
   ```
4. **Giả lập sự cố xóa dữ liệu**: Giả sử quản trị viên chạy nhầm lệnh xóa hoặc xóa nhầm một số bản ghi quan trọng, chỉ giữ lại 2 nhân viên (Linh và Hai Huy):
   ```sql
   DELETE FROM employees WHERE id = 2; -- Xóa nhân viên Chien
   ```
5. Chạy lệnh truy vấn kiểm tra dữ liệu hiện tại:
   ```sql
   SELECT * FROM employees;
   ```
   *Kết quả trên DBeaver chỉ còn lại 2 dòng dữ liệu (id = 1 và id = 4).*

![Dữ liệu bị xóa nhầm chỉ còn lại 2 dòng](../../../../images/aws/rds_lab_aurora_step3_deleted_data.png)

### Bước 4: Thực hiện Backtrack (Quay ngược thời gian)
Thay vì phải restore toàn bộ database từ bản backup snapshot cũ (mất rất nhiều thời gian tạo máy chủ mới và đổi endpoint), chúng ta sẽ sử dụng tính năng Backtrack để quay ngược Cluster về thời điểm vài phút trước khi xảy ra lệnh DELETE:

1. Quay trở lại AWS RDS Console -> mục **Databases**.
2. Tích chọn cụm cơ sở dữ liệu `aurora-cluster-01`.
3. Nhấp chọn menu **Actions** ở góc trên cùng bên phải và chọn **Backtrack**.

![Chọn tính năng Backtrack trong Actions](../../../../images/aws/rds_lab_aurora_step4_backtrack_action.png)

4. Tại giao diện cấu hình Backtrack:
   * Chọn mốc thời gian cụ thể (chọn thời điểm trước khi bạn chạy câu lệnh `DELETE FROM employees WHERE id = 2` khoảng 2-3 phút).
   * Click xác nhận **Backtrack DB cluster**.
   * Hệ thống sẽ chuyển trạng thái cụm sang `Backtracking` và thực hiện khôi phục dữ liệu ở tầng lưu trữ cực nhanh (thường chỉ mất vài giây đến dưới 1 phút).

---

### Bước 5: Kiểm tra và xác nhận dữ liệu đã được khôi phục
1. Đợi cụm chuyển lại trạng thái **Available** trên Console.
2. Quay lại công cụ DBeaver, thực hiện tải lại (Refresh) và chạy lại câu lệnh kiểm tra dữ liệu:
   ```sql
   USE company;
   SELECT * FROM employees;
   ```
3. **Kết quả:** Bản ghi của nhân viên `Chien` (id = 2) đã được khôi phục nguyên vẹn và xuất hiện trở lại trong bảng dữ liệu cùng với các bản ghi khác.
