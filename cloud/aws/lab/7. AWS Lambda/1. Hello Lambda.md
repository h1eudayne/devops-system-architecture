# 1. Hello Lambda (Làm quen với AWS Lambda Console)

Bài thực hành này hướng dẫn bạn từng bước khởi tạo, cấu hình sự kiện kiểm thử (Test Event), viết mã nguồn và thực thi một hàm **AWS Lambda Function** cơ bản sử dụng giao diện quản trị **AWS Management Console**.

---

## I. Mục tiêu bài Lab
* Làm quen với giao diện AWS Lambda Console.
* Khởi tạo thành công một Lambda function đơn giản bằng ngôn ngữ **Python**.
* Cấu hình sự kiện kiểm thử (**Test Event**) để chạy thử hàm.
* Thực thi hàm và kiểm tra kết quả (Response, logs, metrics).

---

## II. Các bước thực hiện chi tiết

### Bước 1: Khởi tạo Lambda Function đơn giản

1. Đăng nhập vào **AWS Management Console**.
2. Tìm kiếm và chọn dịch vụ **Lambda** trên thanh tìm kiếm để truy cập Lambda Dashboard.
3. Nhấp chọn nút **Create function** (Tạo hàm) ở khu vực **Get started**.

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step1_get_started.png" alt="AWS Lambda Get Started" width="650"/>
</p>

4. Tại trang cấu hình tạo hàm:
   * Chọn tùy chọn **Author from scratch** (Tự xây dựng từ đầu).
   * **Function name** (Tên hàm): Nhập `hello-world`.
   * **Runtime** (Môi trường chạy): Chọn **Python 3.13** (hoặc phiên bản Python mới nhất có sẵn).
   * Các thiết lập khác giữ nguyên cấu hình mặc định (AWS sẽ tự động tạo một IAM Execution Role cơ bản để ghi log lên CloudWatch).
5. Nhấp nút **Create function** ở dưới cùng bên phải. Đợi vài giây để AWS hoàn tất thiết lập tài nguyên.

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step2_create_function.png" alt="Thiết lập thông số cho Lambda Function" width="650"/>
</p>

---

### Bước 2: Tạo sự kiện kiểm thử (Create Test Event)

Vì Lambda chạy serverless không có giao diện tương tác trực tiếp, chúng ta cần cấu hình một Test Event giả lập sự kiện gửi payload đến hàm:

1. Tại giao diện quản lý hàm `hello-world`, chọn tab **Test**.
2. Thiết lập thông số:
   * **Test event action**: Chọn **Create new event**.
   * **Event name**: Nhập `hello`.
   * **Event sharing settings**: Chọn `Private` (chỉ bạn mới có quyền chạy test này).
   * **Template - optional**: Chọn `Hello World`.
   * **Event JSON**: Bạn có thể tùy chỉnh JSON gửi đi, ví dụ:
     ```json
     {
       "name": "hieu",
       "age": "18"
     }
     ```
3. Nhấp nút **Save** ở phía trên bên phải để lưu cấu hình sự kiện kiểm thử.

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step3_configure_test.png" alt="Cấu hình Test Event" width="650"/>
</p>

---

### Bước 3: Viết mã nguồn Python và Deploy

1. Quay trở lại tab **Code**.
2. Nhấp đúp mở file `lambda_function.py` trong trình soạn thảo code tích hợp và thay thế nội dung bằng đoạn mã nguồn Python đơn giản dưới đây:
   ```python
   import json

   def lambda_handler(event, context):
       # In dòng chữ "Hello World!" ra log group
       print("Hello World!")
       
       # Trả về mã trạng thái 200 kèm phản hồi
       return {
           'statusCode': 200,
           'body': json.dumps('Hello from Lambda!')
       }
   ```
3. Trong menu xổ xuống bên cạnh nút **Test**, chọn sự kiện kiểm thử `hello` vừa tạo ở Bước 2.
4. **Lưu ý cực kỳ quan trọng**: Mỗi khi sửa đổi mã nguồn hoặc cấu hình, bạn bắt buộc phải nhấp nút **Deploy** để lưu và đẩy phiên bản code mới nhất lên đám mây AWS trước khi tiến hành chạy kiểm thử.

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step4_deploy_code.png" alt="Deploy code mới và chọn Test Event" width="650"/>
</p>

---

### Bước 4: Thực thi kiểm thử và Kiểm tra kết quả

1. Nhấp nút **Test** để kích hoạt hàm Lambda.
2. Tab **Execution result** (Kết quả thực thi) sẽ hiển thị ngay lập tức trong bảng điều khiển phía dưới:
   * **Status**: `Succeeded` (Trạng thái chạy thành công có màu xanh lá cây).
   * **Response**: Trả về đúng dữ liệu phản hồi được cấu hình trong hàm (`'statusCode': 200`, `'body': '"Hello from Lambda!"'`).
   * **Function Logs**:
     - Dòng log `Hello World!` do hàm `print("Hello World!")` in ra hiển thị rõ ràng.
     - Các thông số vận hành như: *Duration* (Thời gian thực hiện thực tế), *Billed Duration* (Thời gian tính tiền), *Max Memory Used* (Dung lượng bộ nhớ tiêu thụ lớn nhất).

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step5_execution_result.png" alt="Kết quả thực thi thành công của Lambda" width="650"/>
</p>

### Bước 5: Xem logs và giám sát trên CloudWatch

Để giám sát lịch sử thực thi và xem nhật ký chi tiết của hàm Lambda, ta sử dụng dịch vụ **Amazon CloudWatch**:

1. Tại giao diện hàm Lambda, chuyển sang tab **Monitor** (Giám sát).
2. Tại đây bạn sẽ thấy các biểu đồ CloudWatch Metrics cơ bản (như Invocations, Duration, Error count). Nhấp chọn nút **View CloudWatch logs** (Xem logs CloudWatch) ở góc phải.

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step6_monitor.png" alt="Giao diện Monitor của Lambda" width="650"/>
</p>

3. Hệ thống sẽ tự động chuyển hướng bạn tới giao diện quản lý log của **CloudWatch Logs** tương ứng với Log Group `/aws/lambda/hello-world`.
4. Trong tab **Log streams**, nhấp chọn **Log stream** mới nhất (được tạo tương ứng với thời gian thực thi hàm).

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step7_cloudwatch_streams.png" alt="Log stream của Lambda" width="650"/>
</p>

5. Tại đây, bạn sẽ thấy toàn bộ nhật ký sự kiện (**Log events**) chi tiết của lần thực thi vừa rồi, bao gồm dòng chữ `Hello World!` được in ra qua hàm `print()` trong mã nguồn cùng báo cáo lượng tài nguyên tiêu thụ chi tiết.

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step8_cloudwatch_events.png" alt="Nhật ký Log events chi tiết trên CloudWatch" width="650"/>
</p>

### Bước 6: Phát hành phiên bản mới (Publish new version)

AWS Lambda hỗ trợ tính năng quản lý phiên bản (Versioning) giúp lưu trữ các snapshot code và cấu hình của hàm tại các thời điểm khác nhau.

1. Tại góc trên bên phải giao diện chi tiết hàm Lambda, nhấn nút **Actions** và chọn **Publish new version** (Phát hành phiên bản mới).

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step9_actions_publish.png" alt="Actions Publish new version" width="650"/>
</p>

2. Nhập mô tả cho phiên bản (ví dụ: `v1.0`) và nhấn **Publish**.
3. **Kết quả**:
   * Phiên bản mới (Version 1) được tạo thành công với một banner thông báo màu xanh lá.
   * Để xem danh sách các phiên bản đã phát hành, bạn chuyển sang tab **Versions** (Phiên bản). Tại đây bạn có thể nhấp chọn trực tiếp từng số phiên bản (Ví dụ: `1`) để xem lại mã nguồn và cấu hình của snapshot đó.

<p align="center">
  <img src="../../../../images/aws/lambda_hello_step10_versions_list.png" alt="Danh sách Versions đã phát hành" width="650"/>
</p>

> [!NOTE]
> * **Lưu ý**: Khi một phiên bản (Version) đã được phát hành, mã nguồn của phiên bản đó sẽ bị khóa và **không thể sửa đổi**. Mọi sửa đổi tiếp theo phải được thực hiện trên phiên bản mặc định `$LATEST` (sau đó mới publish thành phiên bản tiếp theo như Version 2, 3...).
> * Một Lambda function có thể chứa nhiều phiên bản được phát hành song song, giúp phục vụ đắc lực cho các bài toán phân phối lưu lượng (Traffic shifting) hoặc chuyển đổi môi trường mượt mà.
