# CodePipeline, CodeBuild & CodeDeploy

## I. Tổng quan về các dịch vụ CI/CD của AWS

**Nó là gì:**
Đây là bộ ba dịch vụ CI/CD native của AWS:
* **CodePipeline**: Là orchestrator điều phối toàn bộ quy trình CI/CD.
* **CodeBuild**: Là dịch vụ build serverless, chạy các bước build/test dựa trên cấu hình trong file `buildspec.yml`.
* **CodeDeploy**: Tự động hóa việc deploy ứng dụng lên EC2, ECS, Lambda với các chiến lược như rolling, blue/green.

**Khi nào sử dụng:**
* Khi muốn xây dựng pipeline CI/CD hoàn toàn trên AWS mà không cần công cụ bên ngoài.
* Khi đã sử dụng CodeCommit hoặc muốn tích hợp với GitHub/Bitbucket qua AWS.
* Khi cần blue/green deployment cho ECS hoặc Lambda.

---

## II. Hướng dẫn cơ bản để bắt đầu

1. Tạo CodeBuild project với file `buildspec.yml` định nghĩa các bước build, test, push image.
2. Tạo CodeDeploy application và deployment group (nếu deploy lên EC2/ECS).
3. Tạo CodePipeline kết nối các chặng: Source (GitHub/CodeCommit) → Build (CodeBuild) → Deploy (CodeDeploy/ECS).
4. Cấu hình IAM role cho từng dịch vụ với quyền phù hợp.
5. Bật CloudWatch Events/EventBridge để trigger pipeline tự động khi có code push.
