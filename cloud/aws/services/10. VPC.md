# VPC (Virtual Private Cloud)

## I. Tổng quan về Amazon VPC

**Nó là gì:**
VPC là mạng ảo riêng trên AWS, cho phép bạn định nghĩa dải IP, subnet, route table, và kiểm soát lưu lượng mạng. Mỗi tài nguyên AWS (EC2, EKS, RDS) đều chạy trong một VPC. VPC là nền tảng networking cho toàn bộ hạ tầng trên AWS.

**Khi nào sử dụng:**
* Bắt buộc khi triển khai bất kỳ tài nguyên nào trên AWS.
* Khi cần tách biệt môi trường (dev, staging, production) bằng các VPC riêng.
* Khi cần kết nối on-premise với AWS qua VPN hoặc Direct Connect.

---

## II. Hướng dẫn cơ bản để bắt đầu

1. Tạo VPC với CIDR block (ví dụ: `10.0.0.0/16`).
2. Tạo public subnet (có Internet Gateway) và private subnet (có NAT Gateway).
3. Cấu hình route table cho từng loại subnet.
4. Tạo Security Group và Network ACL để kiểm soát traffic.
5. Tạo NAT Gateway trong public subnet để private subnet có thể truy cập internet.
