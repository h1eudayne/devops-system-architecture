# Amazon RDS Hands-on Lab (Cluster)

Bài thực hành này đã được chuyển sang thư mục hướng dẫn triển khai (**deploy**) của dự án để đảm bảo tính phân tách tài nguyên rõ ràng giữa phần tài liệu giới thiệu dịch vụ (Services) và hướng dẫn thực hành triển khai thực tế (Deploy).

Để xem toàn bộ hướng dẫn thực hành từng bước chi tiết (Hands-on Lab) bao gồm:
1. Đăng nhập vào AWS Console và cấu hình khởi tạo cụm cơ sở dữ liệu (RDS DB Cluster) với Multi-AZ DB cluster.
2. Hiểu cấu trúc phân cấp vai trò (Role) của cụm cơ sở dữ liệu: Writer Instance và Reader Instances.
3. Nguyên tắc quan trọng: $\underline{0}$ (KHÔNG) sử dụng các Endpoint riêng lẻ của từng instance để tránh sự cố lỗi kết nối khi xảy ra hoán đổi vai trò (Failover).
4. Sử dụng cố định Cluster Writer Endpoint để thực hiện kết nối ứng dụng cho cả tác vụ đọc ghi.
5. Cấu hình kết nối trên DBeaver thông qua Cluster Write Endpoint để thực thi CRUD.
