# 7. AWS Lambda Use Cases (Các trường hợp sử dụng thực tế)

AWS Lambda được ứng dụng rộng rãi trong nhiều mô hình kiến trúc phần mềm hiện đại nhờ khả năng tích hợp linh hoạt và tính chất serverless tự động co giãn. Dưới đây là 4 nhóm Use Case điển hình nhất trong thực tế.

---

## 1. Tự động hóa xử lý tệp tin và đa phương tiện (File & Media Processing)

Đây là một trong những mô hình kinh điển nhất sử dụng kiến trúc hướng sự kiện (Event-Driven).

```
[Người dùng] ──> Tải ảnh lên S3 Bucket Nguồn
                       │
                       ▼ (Sự kiện ObjectCreated)
                [S3 Event Notification]
                       │
                       ▼ (Kích hoạt)
                [AWS Lambda Function] (Xử lý Resize)
                       │
                       ▼ (Lưu ảnh đã resize)
                 S3 Bucket Đích
```

* **Mô tả luồng**: 
  1. Người dùng upload một bức ảnh có dung lượng lớn lên **S3 Bucket Nguồn**.
  2. Sự kiện tạo mới file kích hoạt trực tiếp hàm **AWS Lambda**.
  3. Hàm Lambda chạy code Python/Node.js để tự động resize (co nhỏ) bức ảnh đó về kích thước tiêu chuẩn, tạo ảnh thumbnail.
  4. Lambda lưu ảnh thumbnail đã xử lý sang **S3 Bucket Đích** để phục vụ hiển thị trên giao diện người dùng.
* **Lợi ích**: Hệ thống không cần duy trì bất kỳ máy chủ nào chạy 24/7 để chờ xử lý file.

---

## 2. Xây dựng Serverless REST APIs / Web & Mobile Backend

Mô hình thiết kế hệ thống backend không máy chủ cho phép ứng dụng mở rộng linh hoạt theo số lượng người dùng truy cập.

```
[Web/Mobile Client] ──> Gửi HTTP Request (POST/GET)
                              │
                              ▼
                     [Amazon API Gateway] (Định tuyến)
                              │
                              ▼ (Chuyển đổi Event)
                     [AWS Lambda Function] (Xử lý Logic nghiệp vụ)
                              │
                              ▼ (Truy vấn dữ liệu)
                    [Amazon DynamoDB / RDS]
```

* **Mô tả luồng**:
  1. Ứng dụng client gửi các HTTP Request (ví dụ đăng ký, đăng nhập, đặt hàng).
  2. **API Gateway** nhận request, thực hiện các bước xác thực cơ bản, định tuyến và chuyển tiếp sự kiện sang **Lambda**.
  3. **Lambda** thực thi code xử lý logic (ví dụ validate dữ liệu, tính toán hóa đơn) và tương tác với cơ sở dữ liệu như **DynamoDB** hoặc **RDS**.
  4. Lambda trả kết quả về cho API Gateway để chuyển đổi thành HTTP Response gửi lại client.

## 3. Tự động hóa vận hành hệ thống (SysOps & Cron Jobs)

Thay thế các máy chủ Jenkins hay Cron Server truyền thống để thực hiện các tác vụ quản trị hệ thống định kỳ.

```
[Amazon EventBridge] (Chạy định kỳ theo Cron Expression: 19:00 hàng ngày)
         │
         ▼ (Kích hoạt)
  [AWS Lambda] (Chạy script kiểm tra và gọi EC2 API StopInstances)
         │
         ▼ (Tắt máy chủ)
  [EC2 Instances] (Cụm máy chủ môi trường Dev)
```

* **Mô tả luồng**:
  1. **Amazon EventBridge (CloudWatch Events)** được cấu hình chạy định kỳ theo lịch (ví dụ: cron job chạy vào lúc 19:00 từ Thứ 2 đến Thứ 6 hàng tuần).
  2. EventBridge kích hoạt hàm **AWS Lambda**.
  3. Hàm Lambda sử dụng thư viện AWS SDK (`boto3` của Python) quét toàn bộ các máy chủ **EC2** trong môi trường Development và tự động tắt chúng để tiết kiệm chi phí qua đêm.

## 4. Xử lý luồng dữ liệu thời gian thực (Real-time Stream Processing)

Phục vụ phân tích dữ liệu lớn và giám sát hoạt động thời gian thực.

```
[Thiết bị IoT / Clickstream] ──> Đẩy dữ liệu liên tục
                                      │
                                      ▼
                              [Amazon Kinesis] (Hứng luồng dữ liệu)
                                      │
                                      ▼ (Kích hoạt theo Batch)
                              [AWS Lambda Function] (Lọc, chuẩn hóa dữ liệu)
                                      │
                                      ▼ (Lưu dữ liệu sạch)
                            [Amazon S3 / OpenSearch]
```

* **Mô tả luồng**:
  1. Hàng ngàn thiết bị IoT hoặc các sự kiện click chuột của người dùng đẩy dữ liệu thô liên tục về **Amazon Kinesis**.
  2. **AWS Lambda** được cấu hình để đọc dữ liệu từ Kinesis theo từng nhóm (batch).
  3. Lambda thực hiện lọc bỏ thông tin rác, chuẩn hóa cấu trúc dữ liệu, và đẩy dữ liệu sạch vào hệ thống lưu trữ **S3** hoặc cơ sở dữ liệu tìm kiếm **Amazon OpenSearch**.
