# 8. Hello Lambda - AWS Console Introduction (Làm quen với AWS Lambda Console)

Bài học này hướng dẫn bạn từng bước khởi tạo, viết mã nguồn, cấu hình kiểm thử và giám sát một hàm **AWS Lambda Function** cơ bản sử dụng giao diện quản trị **AWS Management Console**.

---

## Bước 1: Khởi tạo Lambda Function từ đầu

1. Đăng nhập vào **AWS Management Console**.
2. Tìm kiếm và chọn dịch vụ **Lambda** để mở giao diện điều khiển (Lambda Dashboard).
3. Nhấp chọn nút **Create function** ở góc trên bên phải.
4. Chọn tùy chọn **Author from scratch** (Tự xây dựng từ đầu).
5. Thiết lập các thông số cơ bản cho hàm:
   * **Function name**: Nhập tên hàm (ví dụ: `hello-lambda-function`).
   * **Runtime**: Chọn môi trường chạy code, ví dụ: **Python 3.12** (hoặc phiên bản Python mới nhất có sẵn).
   * **Architecture**: Chọn `x86_64`.
   * **Permissions**: Giữ tùy chọn mặc định *Create a new role with basic permissions*. AWS sẽ tự động tạo một IAM Execution Role cho phép hàm ghi nhật ký logs vào Amazon CloudWatch.
6. Nhấp nút **Create function** ở góc dưới cùng bên phải. Đợi vài giây để AWS thiết lập tài nguyên.

---

## Bước 2: Viết mã nguồn (Code Source)

1. Sau khi giao diện quản lý hàm hiển thị, cuộn xuống tab **Code** và quan sát khu vực **Code source**.
2. Nhấp đúp mở file `lambda_function.py`. Bạn sẽ thấy cấu trúc cơ bản của một hàm Lambda:
   ```python
   import json

   def lambda_handler(event, context):
       # event: Chứa dữ liệu đầu vào gửi tới từ Trigger
       # context: Chứa thông tin về runtime, log stream và thời gian thực thi của hàm
       
       # In dữ liệu event đầu vào để debug
       print("Event data received:", json.dumps(event))
       
       # Trả về kết quả HTTP response mẫu
       return {
           'statusCode': 200,
           'body': json.dumps('Hello from Antigravity AWS Lambda Course!')
       }
   ```
3. Chỉnh sửa code theo ý muốn của bạn, sau đó nhấp nút **Deploy** (phía trên trình soạn thảo code) để lưu và cập nhật mã nguồn lên đám mây AWS.

---

## Bước 3: Cấu hình Sự kiện Kiểm thử (Configure Test Event)

Vì Lambda chạy không có máy chủ trực quan, chúng ta cần tạo một sự kiện giả lập (Test Event) để kích hoạt và chạy thử code:

1. Nhấp chọn nút mũi tên bên cạnh nút **Test** và chọn **Configure test event**.
2. Trong cửa sổ cấu hình:
   * **Event action**: Chọn **Create new event**.
   * **Event name**: Nhập tên sự kiện (ví dụ: `myTestEvent`).
   * **Event sharing settings**: Chọn `Private`.
   * **Template**: Chọn `hello-world`.
   * **Event JSON**: Bạn có thể giữ nguyên JSON mặc định hoặc sửa đổi thông tin truyền vào:
     ```json
     {
       "key1": "value1",
       "key2": "value2",
       "key3": "value3"
     }
     ```
3. Nhấp nút **Save** dưới cùng để lưu lại cấu hình test.

---

## Bước 4: Thực thi kiểm thử và Kiểm tra kết quả

1. Nhấp chọn nút **Test** (vừa cấu hình xong ở Bước 3).
2. Tab **Execution result** hiển thị ngay lập tức với thông tin:
   * **Response**: Kết quả trả về từ hàm (`statusCode: 200` kèm theo chuỗi chữ chào mừng).
   * **Function logs**: Dòng chữ in ra `Event data received: {"key1": "value1", ...}` hiển thị đúng kết quả của hàm `print()` trong code.
   * **Report**: Các thông tin vận hành như:
     - *Duration*: Thời gian thực thi thực tế (ví dụ: `1.20 ms`).
     - *Billed Duration*: Thời gian bị tính tiền (được làm tròn theo đơn vị ms).
     - *Memory Size*: Dung lượng cấu hình bộ nhớ (128 MB).
     - *Max Memory Used*: Dung lượng thực tế hàm tiêu thụ (ví dụ: `36 MB`).

---

## Bước 5: Xem Logs chi tiết trên CloudWatch Logs

1. Chuyển sang tab **Monitor** (Giám sát) của hàm Lambda.
2. Nhấp chọn nút **View CloudWatch logs**.
3. Bạn sẽ được điều hướng tới Log Group `/aws/lambda/hello-lambda-function` trên **Amazon CloudWatch Logs**.
4. Chọn **Log Stream** mới nhất để xem nhật ký logs chi tiết được lưu trữ lâu dài. Đây là nơi bạn sẽ vào để troubleshoot (sửa lỗi) khi ứng dụng thực tế gặp sự cố.
