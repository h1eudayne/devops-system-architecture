# 4. AWS Lambda Integrations (Các dịch vụ kết hợp)

AWS Lambda là một dịch vụ trung tâm (hub) cực kỳ linh hoạt, **có thể liên kết và kết hợp được với gần như tất cả các dịch vụ khác của AWS**, miễn là nó được cấu hình Trigger phù hợp và được cấp **IAM Role** phù hợp.

Khả năng kết hợp này được chia thành hai nhóm chính:

## I. Dịch vụ đầu vào / Nguồn kích hoạt (Event Sources / Triggers)

Hàm Lambda được tự động kích hoạt phản hồi lại các sự kiện từ:
* **Tính toán & Mạng**: **Amazon API Gateway** (xử lý request HTTP/HTTPS từ client), **Application Load Balancer (ALB)**.
* **Lưu trữ**: **Amazon S3** (khi có tệp tin mới được upload hoặc bị xóa).
* **Cơ sở dữ liệu**: **Amazon DynamoDB** (theo dõi thay đổi dữ liệu thời gian thực qua DynamoDB Streams).
* **Truyền tin nhắn & Hàng đợi**: **Amazon SQS** (Message Queue), **Amazon SNS** (Notification), **Amazon EventBridge** (Lập lịch chạy định kỳ hoặc bắt sự kiện hệ thống).
* **Streaming dữ liệu**: **Amazon Kinesis** (phân tích dữ liệu streaming), **Amazon MSK** (Apache Kafka).
* **Thiết bị thông minh**: **AWS IoT**.

## II. Dịch vụ đầu ra / Đích đến (Destinations / Targets)

Sau khi xử lý xong logic, Lambda có thể đẩy kết quả hoặc tương tác với:
* **Ghi nhật ký**: **Amazon CloudWatch** (tự động ghi nhận Logs và Metrics để giám sát hệ thống).
* **Cơ sở dữ liệu**: Ghi và lưu trữ dữ liệu có cấu trúc vào cơ sở dữ liệu quan hệ **Amazon RDS**, hoặc cơ sở dữ liệu NoSQL **Amazon DynamoDB**.
* **Lưu trữ**: Đẩy file output hoặc lưu trữ kết quả xử lý vào **Amazon S3** hoặc **Amazon EFS** (Elastic File System).
* **Dịch vụ thông báo & Hàng đợi**: Gửi email qua **Amazon SES**, hoặc đẩy tin nhắn thông báo qua **Amazon SNS / SQS**.
