# EKS (Elastic Kubernetes Service)

## I. Tổng quan về Amazon EKS

**Nó là gì:**
EKS là dịch vụ Kubernetes được quản lý bởi AWS. AWS sẽ lo phần control plane (API server, etcd, scheduler), bạn chỉ cần quản lý worker node hoặc sử dụng Fargate để chạy pod serverless. EKS tương thích hoàn toàn với Kubernetes open-source nên có thể sử dụng kubectl, Helm, và các tool K8s thông thường.

**Khi nào sử dụng:**
* Khi cần chạy ứng dụng container hóa ở quy mô lớn với khả năng tự động scale.
* Khi team đã quen với Kubernetes và cần managed control plane để giảm chi phí vận hành.
* Khi cần tích hợp sâu với các dịch vụ AWS khác như ALB, IAM, CloudWatch.

---

## II. Hướng dẫn cơ bản để bắt đầu

1. Cài đặt `eksctl` và `kubectl` trên máy local.
2. Chạy lệnh tạo cụm:
   ```bash
   eksctl create cluster --name my-cluster --region ap-southeast-1 --nodegroup-name workers --node-type t3.medium --nodes 2
   ```
3. Cấu hình kubeconfig:
   ```bash
   aws eks update-kubeconfig --name my-cluster --region ap-southeast-1
   ```
4. Kiểm tra cụm:
   ```bash
   kubectl get nodes
   ```
5. Deploy ứng dụng:
   ```bash
   kubectl apply -f deployment.yaml
   ```
