# Amazon IAM Hands-on Lab (Assume Role with AWS CLI)

Bài học thực hành này đã được chuyển sang thư mục hướng dẫn triển khai (**deploy**) của dự án để đảm bảo tính phân tách rõ ràng giữa phần lý thuyết giới thiệu dịch vụ (Services) và hướng dẫn thực hành triển khai thực tế (Deploy).

Để xem hướng dẫn thực hành từng bước chi tiết (Hands-on Lab) bao gồm:
1. Gỡ bỏ User khỏi Group để xóa bỏ quyền hạn trực tiếp.
2. Kiểm tra lỗi `AccessDenied` trên AWS CLI sau khi bị gỡ quyền.
3. Tạo Role `MyPowerRole` với chính sách `PowerUserAccess` và cấu hình Custom Trust Policy cho phép User assume.
4. Chạy lệnh `aws sts assume-role` qua CLI để lấy thông tin xác thực tạm thời (Access Key, Secret Key, Session Token).
5. Cấu hình tệp tin `credentials` với profile phụ `[poweruser]`.
6. Thực hiện kiểm tra thành công quyền truy cập S3 qua profile mới.
