# Amazon IAM Hands-on Lab (AWS CLI and MFA)

Bài học thực hành này đã được chuyển sang thư mục hướng dẫn triển khai (**deploy**) của dự án để đảm bảo tính phân tách rõ ràng giữa phần lý thuyết giới thiệu dịch vụ (Services) và hướng dẫn thực hành triển khai thực tế (Deploy).

Để xem hướng dẫn thực hành từng bước chi tiết (Hands-on Lab) bao gồm:
1. Cài đặt AWS CLI Latest và kiểm tra với `aws --version`.
2. Phát hành Access/Private Key cho User an toàn.
3. Thiết lập Access/Secret Key cho CLI (`aws configure`), kiểm tra thông tin tài khoản (`aws sts get-caller-identity`) và một số câu lệnh tương tác cơ bản với S3.
4. Tạo policy `enforce-mfa-policy` và gắn vào Group (Admin) để bắt buộc xác thực đa yếu tố.
5. Kiểm tra lại CLI sau khi gán Policy (Expect Access Denied).
6. Kích hoạt thiết bị xác thực MFA ảo cho User trên AWS Console.
7. Lấy Session Token tạm thời qua CLI (`aws sts get-session-token`) sử dụng MFA ARN của thiết bị (không phải ARN của User) và cấu hình tệp tin `credentials` cho profile `[mfa]`.
8. Sử dụng các lệnh AWS CLI kèm theo `--profile mfa` để truy cập thành công tài nguyên.
